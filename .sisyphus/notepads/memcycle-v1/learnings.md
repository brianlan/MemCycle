- Tauri v2 menubar pattern uses `TrayIconBuilder` in Rust setup and toggles a hidden panel window via `get_webview_panel("main")`; Dock icon is hidden with `ActivationPolicy::Accessory` in app setup.
- `tauri.conf.json` keeps the panel hidden initially (`visible: false`) with transparent, undecorated window config and enables `macOSPrivateApi` for macOS panel behavior.
- Menubar panel auto-hide is implemented with NSWorkspace notifications plus panel resign-key events, using Cocoa/ObjC interop from `tauri-nspanel`.
- 2026-02-21: Tauri v2 scaffold with `bun create tauri-app@latest` creates a default project with `opener` plugin, which adds `opener:default` to capabilities. When you remove the plugin (to use sql plugin instead), this permission becomes invalid and causes build failure.
- 2026-02-21: When switching plugins in Tauri v2, must update both Cargo.toml and capabilities/default.json to remove old plugin permissions.
- 2026-02-21: Rust/Cargo must be sourced via `. $HOME/.cargo/env` before running `tauri` commands in this environment.
- 2026-02-21: TypeScript types/conventions: timestamps use ISO 8601 strings (not Date objects), IDs are strings (UUIDs from DB), algorithm is union type ("sm2" | "leitner"), rating is 1-4 integer literal type.
- 2026-02-21: SQLite migrations in Tauri v2 use `tauri_plugin_sql::Builder::add_migrations("sqlite:app.db", migrations)` where migrations is a `Vec<Migration>`. Each Migration has version, description, sql (via `include_str!`), and kind (Up/Down).
- 2026-02-21: tauri.conf.json plugins config goes at root level (not under `app`), e.g., `{"plugins": {"sql": {"preload": ["sqlite:app.db"]}}}`.
- 2026-02-21: DB path: SQLite database created at app data directory (e.g., `~/Library/Application Support/com.memcycle.flashcard/app.db` on macOS). To verify tables: `sqlite3 ~/Library/Application\ Support/com.memcycle.flashcard/app.db ".tables"`.
- 2026-02-21: Types align with DB schema: Card has deckId, CardSchedulingState maps all card_scheduling columns, ReviewLog has elapsedMs, Settings is key-value store.

- 2026-02-21: Leitner convention is 0-based `boxIndex` mapped directly to `BOX_INTERVALS` (`boxIndex` 0 == box 1). Correct answer moves `boxIndex + 1` capped at max index; incorrect resets to 0. Compute `dueAt` from the resulting `boxIndex` interval, and keep timestamps as ISO strings.
- 2026-02-21: SM-2 rating mapping chosen for 1-4 UI scale is `1->2`, `2->3`, `3->4`, `4->5` on SM-2 q-scale; this keeps rating 1 as lapse (`q < 3`) while ratings 2-4 remain successful reviews.
- 2026-02-21: SQLite foreign keys are connection-scoped. Cascade delete works only when `PRAGMA foreign_keys = ON` is set for that connection (enabled in deck repository DB init; also needed in sqlite3 CLI verification).
- 2026-02-21: Card repository scheduling init chooses Leitner `interval_days = BOX_INTERVALS[0]` (1 day) instead of schema default 0, to keep repository-created Leitner cards aligned with algorithm interval semantics from first box.
- 2026-02-21: No-response daily windows are computed in UTC using `[reviewed_at >= dayStartUtc, reviewed_at < nextDayStartUtc)` so daily caps reset naturally at UTC midnight with no background job.
- 2026-02-21: Await-session representation for no-response cap uses `card_scheduling.due_at = nextDayStartUtc` once a card reaches 3 no-responses in the current UTC day; `shouldShowCard` still enforces the 3/day gate directly from review_logs.
- 2026-02-21: Scheduling service Leitner review mapping uses `rating >= 3` as correct and `rating <= 2` as incorrect; lapse count increments only on incorrect Leitner reviews.
- 2026-02-21: Daily reviewed count and streak both use UTC day boundaries `[dayStartUtc, nextDayStartUtc)`, and streak is consecutive UTC calendar days including today only when today has at least one rated review.
- 2026-02-21: Tauri v2 tray support is feature-gated; enable `tauri` crate feature `tray-icon` in `src-tauri/Cargo.toml` or `tauri::tray` imports fail to compile.
- 2026-02-21: Native tray menu wiring pattern: build `MenuItem::with_id(...)`, attach via `TrayIconBuilder::menu(...)`, handle clicks with `on_menu_event` + `AppHandle::emit`, and use `on_tray_icon_event` to emit a left-click event without implementing dropdown UI yet.
- 2026-02-21: Tauri popup window can be created lazily with `WebviewWindowBuilder` using `transparent(true)`, `decorations(false)`, `always_on_top(true)`, `skip_taskbar(true)`, and `visible(false)` to behave like a tray-style overlay window without adding extra plugins.
- 2026-02-21: Popup event emission pattern: emit `popup:shown` after show, `popup:dismissed` after manual/timeout hide, and `popup:timeout` on auto-dismiss; nonce in shared `State<Mutex<u64>>` prevents stale timeout tasks from hiding newly-shown popups.
- 2026-02-21: Settings encryption pattern for `llmApiKey`: use AES-GCM payload `enc:v1:<iv-b64>:<cipher-b64>` in DB and store raw AES key (base64) outside DB at `localStorage['memcycle.settings.encryption-key.v1']`.
- 2026-02-21: Caveat for settings service tests/CLI scripts: `@tauri-apps/plugin-sql` depends on Tauri window runtime, so `setSetting/getSetting` cannot be executed in plain Bun/Node without a Tauri webview context.
- 2026-02-21: Timer persistence approach for Task 17: Rust timer emits `timer:next_fire_updated` after start/reset/fire, and TS `timerService` persists `nextFireTimestampMs` into `settings.timerNextFireTimestampMs`; on app restart, `startTimer()` reads that value and passes it back to Rust `start_timer` for restart recovery.
- 2026-02-21: Task 19 export/import JSON format fixed at `version: "1.0"` with top-level `exportedAt` ISO string, `cards` embedding full `scheduling` state, optional `reviewLogs`, and `settings` excluding `llmApiKey`.
- 2026-02-21: Task 19 import strategy validates first, then runs a single SQL transaction; default behavior replaces existing decks/cards/scheduling/review logs and non-sensitive settings only after validation passes, while resolving deck-name collisions by suffixing ` (imported)` (and numeric suffixes as needed).
- 2026-02-21: Task 19 format update changed `settings` from an array of `{ key, value }` to a JSON object map `{ [key]: value }`; validation now enforces object shape and string-only values while rejecting `llmApiKey`.
- 2026-02-21: Task 19 DB access now goes through `remoteDb.getDatabase()` so export/import respects active local vs remote mode used by repositories/services.
- 2026-02-21: PostgreSQL support for Tauri SQL plugin is enabled by adding `postgres` in `tauri-plugin-sql` features (`src-tauri/Cargo.toml`) alongside `sqlite`.
- 2026-02-21: Remote DB mode switching is safer when all DB consumers share one loader (`src/lib/repositories/remoteDb.ts`), so `switchToRemote()/switchToLocal()` can reset one cache and affect repositories/services consistently.
- 2026-02-21: Added Postgres DDL at `src-tauri/migrations/pg/0001_init.up.sql`; schema stays aligned with SQLite and uses `DOUBLE PRECISION` for `ease_factor` on Postgres.
- 2026-02-21: For Task 18 verification, domain data modules were confirmed to use shared loader (`remoteDb.getDatabase()`): deck/card repositories plus scheduling/no-response services; settings/export remain intentionally separate for scope control.
- 2026-02-21: Timer units clarification: `nextFireTimestampMs` persistence/event payloads remain milliseconds since epoch, but `get_time_until_next` returns remaining whole seconds (ceiling) for UI-friendly countdown semantics.
- 2026-02-21: React keyboard handling pattern for review UIs: Use `useEffect` with `window.addEventListener('keydown')` for global shortcuts (Space, 1-4) instead of onFocus inputs, ensuring `e.preventDefault()` on Space to block scrolling. Clean up listeners in return callback.
- 2026-02-21: Bun test vs Vitest discovery: Renamed component tests needing DOM to `*.vitest.tsx` and updated `vitest.config.ts` to include them, preventing `bun test` (which lacks jsdom) from failing on them while keeping fast unit tests running in both.
- 2026-02-21: Handling Cmd+Enter in Textareas: Use `useEffect` with `keydown` listener on textarea refs for reliable `Cmd+Enter` (or `Ctrl+Enter`) support, as `onKeyDown` prop might not always capture shortcuts cleanly across all browsers/OS without preventing default behavior explicitly. Ensure to check `(e.metaKey || e.ctrlKey) && e.key === "Enter"` to support both macOS and Windows/Linux.
- 2026-02-22: E2E Testing Locator Strategy: `shadcn/ui` components like `CardTitle` and `DialogTitle` often render as `div` elements despite semantic naming. Use `getByText(..., { exact: true })` or scoped locators (`[data-testid="..."]`) instead of `getByRole('heading')` to avoid test flakiness, unless confirming the underlying HTML tag is necessary.
- 2026-02-22: `crypto.randomUUID()` in Playwright/Vite Test Environment: When running E2E tests against a local dev server (http://localhost), `crypto.randomUUID()` might be undefined or behave unexpectedly in some contexts. Using a robust fallback (e.g. `crypto.randomUUID ? crypto.randomUUID() : Math.random()...`) ensures ID generation works reliably during tests.
