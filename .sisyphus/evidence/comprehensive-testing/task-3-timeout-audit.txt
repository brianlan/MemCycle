### Files Audited and Refactored

#### 1. e2e/infrastructure/wait-helpers.ts (NEW FILE)
Created reusable wait utilities with the following functions:
- `waitForNetworkIdle()` - Wait for network to be idle after navigation/API calls
- `waitForElementStable()` - Wait for elements to stop animating/transitioning
- `waitForDropdownOptions()` - Wait for dropdown options to populate (replaces waitForTimeout)
- `waitForSelectionToRegister()` - Wait for selection to register (replaces waitForTimeout)
- `waitForElementCount()` - Wait for specific count of elements (replaces waitForTimeout)
- `waitForElementReady()` - Wait for element to be visible and enabled
- `waitForTextContent()` - Wait for text content to appear (replaces waitForTimeout)
- `waitForOverlayDismissed()` - Wait for modal/overlay to be dismissed
- `retryAction()` - Retry a specific action until it succeeds
- `isElementCovered()` - Check if element is covered by an overlay
- `forceClickWithJustification()` - Force click with automatic documentation

### Anti-Patterns Addressed

1. **Arbitrary timing waits** → Replaced with:
   - `expect(locator).toBeVisible()` - wait for visibility
   - `expect(locator).not.toBeVisible()` - wait for dismissal
   - `waitForDropdownOptions()` - wait for dropdown population
   - `waitForSelectionToRegister()` - wait for selection registration
   - `waitForElementReady()` - wait for element to be interactive

2. **Undocumented force clicks** → Added comments explaining:
   - Validation overlay covering submit button
   - Hover overlay covering action buttons
   - Modal overlay covering confirm button
   - Loading overlay covering generate button

3. **Silent error handling** → None found (all errors properly handled)

### Flakiness Prevention Guidelines Added

Each helper function in `wait-helpers.ts` includes usage guidelines:

1. NEVER use waitForTimeout() - always use condition-based waits
2. Prefer expect().toBeVisible() over waitForTimeout()
3. Use expect(async () => {}).toPass() for retry logic with assertions
4. Use waitForNetworkIdle() only for page transitions and API calls
5. Use waitForElementStable() for elements that may be in transition
6. If force: true is required, ALWAYS document why
